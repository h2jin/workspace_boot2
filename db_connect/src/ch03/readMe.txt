
preparedStatement 사용방법
: statement와 preparedStatement의 차이점은 캐시 사용 유무이다.

캐시란? 
사용자가 바나나를 찾고 싶은 경우, 동작은 메모리(램)에 바나나라는 데이터가 있는지 찾고,
 없다면 하드디스크에 가서 서치함. 
 바나나 데이터가 있으면 가져와서 메모리에 올린다음 사용자에게 전달.
 
 사용자가 또 바나나를 서치하는 경우 하드디스크에서 서치할 필요없다, 메모리에 있기 때문. 
 이를 캐시한다 라고 함...
 
 따라서 반복적으로 커리를 수행한다면 Statement에 비해 성능이 훨씬 좋다.
 그리고 보안적인 측면에서도 Statement보다 PreparedStatement 가 안정성이 높다.
